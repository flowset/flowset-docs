= Setup

[[check-system-requirements]]
== System Requirements

Before proceeding with installation, ensure your system meets the following minimum requirements:

* Operating System: 64-bit Windows, Linux, or macOS
* RAM: Minimum 8 GB (16 GB recommended)
* Disk Space: Minimum 10 GB of free space
* Java: JDK or JRE version 21

Verify your Java installation by running:

[source,shell]
----
java -version
----

== Installation Methods

[[portable]]
=== Portable Distribution

The portable distribution allows you to run OpenBPM Control without additional dependencies (except Java). Ideal for quick testing or local development.

*Running Instructions*:

. Download the portable archive from the provided https://openbpm.io/downloads#control[link^].

. Extract the archive to your preferred directory.

. Execute the launch script based on your OS:
+
*Linux/macOS*:
+
[source,shell]
----
./run.sh
----
+
Windows:
+
[source,bash]
----
run.bat
----
. Access the application at: `++http://localhost:8081++`.

[[docker-compose]]
=== Docker Compose

This method allows for quick startup of OpenBPM Control in containers, without manual database or environment configuration.

*Prerequisites*:

* *Docker* and *Docker Compose*
* Verify installations:
+
[source,shell]
----
docker --version
docker compose version
----

*Running Instructions*:

. Create a `docker-compose.yml` file with:
+
[source,yaml]
----
version: '3.8'

services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: openbpm-control
      POSTGRES_USER: root
      POSTGRES_PASSWORD: root
    ports:
      - "5432:5432"

  openbpm-control:
    image: openbpm/openbpm-control:latest
    environment:
      MAIN_DATASOURCE_URL: jdbc:postgresql://postgres:5432/openbpm-control
      MAIN_DATASOURCE_USERNAME: root
      MAIN_DATASOURCE_PASSWORD: root
      SERVER_PORT: 8081
    ports:
      - "8081:8081"
    depends_on:
      - postgres
----

. Start the containers:
+
[source,shell]
----
docker compose up -d
----

. Access the application at: `++http://localhost:8081++`.

. To stop the containers:
+
[source,shell]
----
docker compose down
----

Detailed instructions are available at https://github.com/openbpm-platform/openbpm-control/tree/main/docker-compose[GitHub^].

[[source-code-build]]
=== Source Code Build

Build and run from source for development or custom deployments.

*Prerequisites*:

* Git
* Java 21 JDK
* PostgreSQL database

*Running Instructions*:

. Clone the repository:
+
[source,shell]
----
git clone https://github.com/openbpm-platform/openbpm-control
----
. Configure PostgreSQL database.
+
OpenBPM Control stores data, such as BPM engine connections, in a database. By default, OpenBPM Control requires a PostgreSQL database with the following parameters:

* Database name: `openbpm-control`
* Username: `root`
* Password: `root`

. Navigate to the project directory and run:
+
[source,shell]
----
./gradlew bootRun
----

. Access the application at: `++http://localhost:8081++`.

[[post-installation]]
== Post-Installation

After successful installation:

. Open your web browser and navigate to `++http://localhost:8081++`.
. Use default administrator credentials (`admin`/`admin`).
. Configure your xref:connecting-engine.adoc[BPM engine] connections.
