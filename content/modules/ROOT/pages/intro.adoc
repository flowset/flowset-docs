= Introduction

== What is OpenBPM?

OpenBPM is a business process automation platform with its own execution engine and a set of tools covering the entire BPM solution lifecycle.

=== Core Principles

* **Compatibility with Camunda**
  We believe this technology, which has become an industry standard, will last for a long time.
  We provide our own engine, registered in the Ministry of Digital Development registry, API-compatible with Camunda 7.

* **Adherence to the BPMN 2.0 Standard**
  Ensures transparency, predictability, and facilitates talent acquisition.

* **Interoperability**
  OpenBPM is compatible with popular modeling and development tools.

* **Focus on Business Process Automation**
  OpenBPM covers the full cycle of process automation—from design to execution and monitoring.

* **Modularity and Openness**
  Each component of the platform works independently, and together they deliver maximum value.

=== Platform Components

==== Engine
A process execution engine, API-compatible with Camunda 7. Currently in the process of being added to the Ministry of Digital Development registry (application submitted).
An open-source version is available on Gitflic.

==== Control
An open-source tool for monitoring business process execution. Similar to Camunda Cockpit but with extended functionality.
Built on the open-source Jmix framework. Source code available on Gitflic.

==== Studio
A plugin for IntelliJ IDEA/OpenIDE. Adds a BPMN/DMN editor directly into the development environment, along with additional capabilities for code generation, navigation, etc.

==== Tasklist (in development)
A React-based application template for managing end-user tasks.

==== Workspace (in development)
A web application for prototyping BPMN diagrams and forms.
